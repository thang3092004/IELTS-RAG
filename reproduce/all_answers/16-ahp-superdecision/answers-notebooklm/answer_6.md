When there are sub-criteria, they should be connected to the alternatives, not the main criteria. The sub-criteria should be placed in their own cluster. For example, a cluster could be named "sub-criteria for research".  The cluster can be color-coded for clarity. 

**Here are the steps for incorporating sub-criteria into the AHP hierarchy:**

*   **Deselect the connection between the main criteria and the alternatives:** Right-click on the main criteria and select "node connection form." Then, deselect the alternatives.
*   **Create a cluster for the sub-criteria:** Click "design" then "cluster." Name the cluster (e.g., "sub-criteria for research"). Save the cluster.
*   **Create nodes for the sub-criteria:** Click "design" then "node" then "new." Create a node for each of the sub-criteria. Save the nodes.
*   **Connect the sub-criteria cluster to the main criteria:** Click on the main criteria, then select "node connection form." Select the sub-criteria nodes and click OK. 
*   **Connect each sub-criterion to the alternatives:** Click on a sub-criterion, then "design" then "node connection form." Select all the alternatives and click OK. Repeat for each sub-criterion. 
*   **Form a pairwise comparison matrix for the sub-criteria:** Click on the main criteria, then click "assess" then "compare" then "pairwise comparisons". Enter the judgments to compare the sub-criteria with respect to the main criteria. 
*   **Compare the alternatives with respect to each sub-criterion:** Click on a sub-criterion, then click "assess" then "compare" then "pairwise comparisons." Enter judgments to compare the alternatives with respect to the sub-criterion. Repeat for each sub-criterion.
*   **Synthesize the model to get the global priorities:** Click "computations" then "synthesize the whole model".

This process can be used to incorporate multiple levels of sub-criteria. For instance, in a model for nuclear waste disposal, sub-criteria are nested within criteria, with the alternatives at the bottom level of the hierarchy.
