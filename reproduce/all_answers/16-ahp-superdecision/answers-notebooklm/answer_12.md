The process of creating an Analytic Hierarchy Process (AHP) hierarchy in SuperDecisions software involves several steps:

### Defining the Goal

*   The first step is to define the goal of the decision-making process. 
*   This is the overall objective that you are trying to achieve. 
*   In SuperDecisions, you create a **goal cluster**, which will contain a **goal node** representing the specific aim. For instance, if the goal is to select the best job, you would create a goal cluster labeled "Selection of the Best Job" and a node within that cluster also labeled "Goal."

### Defining the Criteria

*   The next step is to define the criteria that will be used to evaluate the alternatives. 
*   These are the factors that are important to you in making the decision. 
*   In SuperDecisions, you create a **criteria cluster**, which will contain **nodes** representing each criterion. For example, criteria for selecting the best job might include salary, research opportunities, growth potential, working environment, location, and reputation. Each of these would be represented by individual nodes within the criteria cluster.

### Defining the Alternatives

*   The final step is to define the alternatives that you are considering. 
*   These are the different options that you are choosing from. 
*   In SuperDecisions, you create an **alternatives cluster**, which will contain **nodes** representing each alternative.  For example, if you are choosing between three job offers from companies A, B, and C, each company would be represented by a separate node within the alternatives cluster.

### Connecting the Elements

Once you have defined the goal, criteria, and alternatives, you need to connect them in a hierarchical structure.  In SuperDecisions, this is done by linking the nodes in the different clusters:

*   The **goal node** is connected to each of the **criteria nodes**. This shows that the criteria are the factors that will be used to evaluate how well each alternative meets the goal.
*   Each **criterion node** is then connected to each of the **alternative nodes**. This shows that each alternative will be evaluated against each criterion.

This hierarchical structure provides a visual representation of the decision-making problem and the relationships between the goal, criteria, and alternatives.
