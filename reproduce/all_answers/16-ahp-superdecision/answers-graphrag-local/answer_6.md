## Incorporation of Sub-Criteria in AHP Hierarchy Using Super Decisions Software

To effectively incorporate sub-criteria into the Analytic Hierarchy Process (AHP) hierarchy within the Super Decisions software, users can follow a systematic approach outlined below. This approach focuses on the structure, evaluation, and the analytical processes facilitated by the software.

### Structuring the AHP Model

1. **Define the Main Goal**:
   At the top of the hierarchy, clearly articulate the main objective or goal you wish to achieve through the decision-making process. For instance, if the goal is to select the best vehicle, this will be the focal point of the hierarchy.

2. **Identify Criteria**:
   Determine the primary criteria that are essential for achieving the main goal. Using the vehicle selection example, criteria might include Price, Comfort, Fuel Efficiency, and Prestige.

3. **Establish Sub-Criteria**:
   Under each primary criterion, identify relevant sub-criteria that further specify the factors to be considered. For Price, sub-criteria could include Initial Cost, Maintenance Cost, and Insurance Rates. This multi-layered approach enhances precision in evaluating alternatives.

### Creating the AHP Model in Super Decisions

1. **Use the Software Interface**:
   Open Super Decisions and utilize the modeling interface to set up your hierarchy. The software allows you to create a "Goal Node" at the top, followed by the main criteria nodes. Each primary criterion can then have corresponding sub-criteria nodes linked to it.

2. **Add Nodes for Sub-Criteria**:
   When defining each primary criterion, users can add new nodes that represent sub-criteria. The software interface supports creating relationships, which allows you to visualize how sub-criteria relate back to their parent criteria.

3. **Link Criteria and Sub-Criteria**:
   Establish graphical connections within the software that illustrate how each sub-criterion is tied to its respective main criterion. This is done through node connections in the Super Decisions user interface, ensuring a structured and organized hierarchy.

### Evaluating Alternatives and Judgments

1. **Input Judgments**: 
   After structuring your criteria and sub-criteria, users must input their judgments regarding the importance of each criterion relative to the overall goal. Super Decisions enables pairwise comparisons that can include comparisons among sub-criteria explicitly to assess their relative importance.

2. **Perform Sensitivity Analysis**:
   Utilize the sensitivity analysis features of Super Decisions to understand how changes in the weightings of sub-criteria affect the overall prioritization of alternatives. This feature allows you to observe how ranking changes based on varying priorities assigned to different factors.

3. **Synthesize Results**:
   Finally, after completing the evaluations based on your structured hierarchy of criteria and sub-criteria, use the software’s synthesis function to aggregate the results. This function can compute overall priorities for each alternative derived from the assessments and judgments provided.

### Conclusion

In summary, Super Decisions software facilitates the careful incorporation of sub-criteria into the AHP hierarchy by allowing users to define, structure, evaluate, and analyze decision-making factors systematically. Through establishing clear relationships and using the various functionalities of the software, users can enhance the granularity of their models, leading to more informed and precise decision-making outcomes.
