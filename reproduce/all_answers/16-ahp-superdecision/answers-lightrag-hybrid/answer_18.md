## Defining and Utilizing Rating Scales in SuperDecisions

In SuperDecisions, effectively creating and using rating scales is crucial for implementing a Ratings Model. This structured approach facilitates a comprehensive evaluation of alternatives based on predetermined criteria. Here’s a detailed overview of how to define and utilize these rating scales in SuperDecisions.

### Step 1: Selecting Criteria

The first step involves identifying and selecting the specific criteria that you'll be assessing. In the context of a Ratings Model, criteria could encompass various aspects like *Salary*, *Growth*, *Location*, and *Research*. Each of these criteria will serve as a parameter against which the alternatives are evaluated. Users actively navigate through these criteria within the SuperDecisions interface to set the groundwork for the rating scales.

### Step 2: Creating Rating Scales

Once the criteria are identified, you can create rating scales tailored to each criterion. In SuperDecisions, users can do this by utilizing the "Add New" option within the Ratings Table. Here, you can define your rating scale using terminology that fits your assessment context, such as *Excellent to Poor*, or introduce custom scales like *Luxurious, Very Good, and Average*. 

Users need to specify scale values associated with each level of the scale, which provide quantitative scores for qualitative assessments. For instance, assigning numerical values could look like this:

- Excellent: 5
- Very Good: 4
- Good: 3
- Average: 2
- Poor: 1

### Step 3: Loading and Managing Existing Scales

In the SuperDecisions environment, users have the option to *load scales from files* if pre-defined rating scales are available. This feature allows for a more streamlined process when working with established scales and eliminates the need to redefine scales previously established in other evaluations. The user retains the flexibility to manage these scales to fit current analytical needs by adjusting, renaming, or reassigning values.

### Step 4: Implementing Pairwise Comparisons

After defining the rating scales, it’s time to implement the pairwise comparisons using the established scales. By comparing two alternatives at a time based on the various criteria you’ve defined, users can obtain actual scores that reflect their preferences. This is an essential process, as it allows users to prioritize choices systematically and confirms the effectiveness of the defined rating scales in practice.

### Step 5: Analyzing Results and Updating Scales

Once comparisons are made and results are generated, users can analyze the outcomes through the Ratings Table or comparison matrices. This process reveals how different alternatives stack up against one another based on the established scales, allowing for informed decisions. If inconsistencies or unexpected results arise, review and update the scales accordingly. SuperDecisions also provides an *Inconsistency Metric* that can guide users in refining their judgments.

### Conclusion

Defining and utilizing rating scales in SuperDecisions involves a methodical process starting from criterion selection to the actual comparison and analysis of alternatives. This structured approach ensures that users can analyze decisions comprehensively while allowing the flexibility to adapt the scales and criteria as needed. By following these steps, users can harness the full capabilities of the SuperDecisions software, enabling them to make informed, data-driven decisions in their evaluations and comparisons effectively.
