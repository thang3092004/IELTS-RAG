Anthropic's prompt caching with the Gemini 1.5 model uses a "cloud 3.5 sonnet" API, which allows for up to 3000 tokens per API call. This is a cost-efficient option compared to Google's context caching, which only allows 32000 tokens per API call. However, Google's context caching provides a latency reduction benefit. If the user's prompt is not too long, using Gemini 1.5 for the first API call and then switching to Google's context caching for subsequent calls can result in lower latency. On the other hand, if the prompt is very long, Google's context caching may be a better option. In the case of a 300 token prompt, Gemini 1.5's context caching can be more cost-efficient.
