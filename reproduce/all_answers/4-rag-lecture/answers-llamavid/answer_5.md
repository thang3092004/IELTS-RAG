The Agentic RAG framework allows for iterative refinement of the query to generate a more relevant and accurate response. The process starts with the user inputting a question. The query is then passed through the RAG pipeline, where it is split into chunks based on the chunking strategy. The chunks are then passed to the LLM model, which generates contextual information. The contextual information is then passed through the embedding models, which generate embeddings. The embeddings are then used to generate the final response.

The query refinement process can be iterative, with the user providing feedback on the response to further refine the query. For example, if the initial response was too general, the user can provide more specific details about what they are looking for. The updated query is then passed through the RAG pipeline again, where the updated contextual information is used to generate a new response. This process can be repeated until the user is satisfied with the final response.

In summary, the Agentic RAG framework allows for iterative refinement of the query to generate a more relevant and accurate response, with the user providing feedback to refine the query and the RAG pipeline generating a new response based on the updated query.
