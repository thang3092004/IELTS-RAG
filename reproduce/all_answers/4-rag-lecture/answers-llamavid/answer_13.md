GraphRAG is a framework that combines knowledge graphs with retrieval augmented generation or RAG. The goal is to address some of the limitations of the current RAG systems. The key components of GraphRAG are:

1. Chunking: The documents are split into chunks based on context. This allows the models to focus on the most relevant information for each query.

2. Embeddings: The chunks are converted into vector representations using embedding models. This allows the models to focus on the most relevant information for each chunk. 

3. Contextualization: The embeddings are used to create context for each chunk. This allows the models to focus on the most relevant information for each community or level of information.

4. GraphRag: The chunks are combined using the graphrag system to generate an answer. This allows the models to focus on the most relevant information for the query.

5. Customization: The contextualization strategies and number of chunks can be customized based on the specific needs of the application.

In summary, GraphRAG provides a powerful framework for building knowledge-based RAG systems by combining knowledge graphs with RAG. This allows for more efficient and accurate retrieval of relevant information for the user's query.
