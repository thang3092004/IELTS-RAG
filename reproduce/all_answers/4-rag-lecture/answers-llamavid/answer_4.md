Traditional RAG and Agentic RAG represent two different approaches to building a retrieval-augmented generation system. Traditional RAG focuses on building a local model and then using a language model to generate a response. In contrast, Agentic RAG introduces agents into the system to help improve the user query before passing it to the language model. 

The key difference is that Agentic RAG introduces agents into the system before the language model. These agents help reformulate the user query to make it semantically close to the target. This helps improve the accuracy of the response generated by the language model.

In Agentic RAG, an agent is responsible for:

- Re-formulating the user query
- Re-ranking the query to make it more relevant
- Adding or removing context to the query

This helps improve the accuracy and relevance of the response generated by the language model. Without the agents, the language model would generate responses that are not close enough to the user's actual question.

In summary, the agents help reformulate the user query in a way that makes it semantically close to the target, improving the accuracy and relevance of the generated response.
