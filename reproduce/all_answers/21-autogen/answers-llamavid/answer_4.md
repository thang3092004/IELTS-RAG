AutoGen enables complex LLM-based workflows by using multi-agent conversations to orchestrate tasks that require multiple skills and knowledge. For example, in the workflow for finding cheap flights, it uses a user proxy agent to request the data from the API, an assistant agent to fetch the data, and a data engineer to generate the SQL query. The user proxy agent then passes the data to the assistant agent, which runs the query and returns the results. The assistant agent then passes the results to the data engineer, which generates the SQL query and returns the result. Finally, the data engineer passes the result to the user proxy agent, which sends it to the human analyst for review. This workflow requires multiple agents with different skills and knowledge to work together seamlessly. AutoGen provides a framework for building and deploying such workflows by enabling the user proxy agent to manage the conversation between the agents, passing data between them, and providing feedback to the agents.
