Data schema is created within Neon to specify the data that needs to be fetched from an API. The schema is defined by the data engineer within the AutoGen workflow. The data schema includes details like the name of the API, the data fields, the data format, and any constraints on the data. The data schema is then passed to the data retriever agent, which uses the schema to make API calls to the API and fetch the data. The data schema is also passed to the assistant agent, which uses the schema to analyze the data and generate insights. The data schema is then passed to the manager agent, which uses the schema to orchestrate the workflow process.
