In an AutoGen workflow, the Amadeus flight server is utilized by the agents to query flight data and generate flight recommendations for the user. The user proxy agent initiates the workflow by making a request for flight recommendations. The flight manager agent then retrieves flight data from the Amadeus flight server using the API key and stores it in a database. The flight manager agent then passes the flight data to the data retriever agent, which uses the data to generate SQL queries to query the Amadeus flight server. The flight manager agent then passes the flight data to the senior data analyst agent, which analyzes the data and returns a flight recommendation. The flight manager agent then passes the flight recommendation to the user proxy agent, which returns it to the user.
